# Prisma Setup Guide

This guide walks through setting up a new Node.js project with **TypeScript**, **Prisma ORM**, and a **Neon PostgreSQL** database.

---
## Documentation
```
https://github.com/CodeScrapper1/prisma-course/blob/main/prisma-complete-documentation-light.pdf
```

# 1. Create a New Node.js Project

Initialize a new Node.js project.

```bash
npm init -y
```

---

# 2. Install TypeScript

Install TypeScript along with the required development dependencies.

```bash
npm install typescript ts-node @types/node --save-dev
```

---

# 3. Create a TypeScript Configuration

Generate the `tsconfig.json` file.

```bash
npx tsc --init
```

---

# 4. Install Prisma CLI

Install Prisma as a development dependency.

```bash
npm install prisma --save-dev
```

---

# 5. Install Prisma Client

Install the Prisma Client package.

```bash
npm install @prisma/client
```

---

# 6. Initialize Prisma

Create the Prisma configuration files.

```bash
npx prisma init
```

This command creates the following structure:

```text
prisma/
└── schema.prisma

.env
```

---

# 7. Create a PostgreSQL Database

1. Create an account on **Neon**.
2. Create a new PostgreSQL project.
3. Copy the database connection string.

Example:

```env
DATABASE_URL="postgresql://username:password@hostname/database?sslmode=require"
```

---

# 8. Configure Environment Variables

Open the `.env` file generated by Prisma.

Replace the existing `DATABASE_URL` with your Neon PostgreSQL connection string.

```env
DATABASE_URL="postgresql://username:password@hostname/database?sslmode=require"
```

| typescript | TypeScript compiler |
| ts-node | Run TypeScript directly |
| @types/node | Node.js type definitions |
| prisma | Prisma CLI |
| @prisma/client | Prisma Client for database access |
